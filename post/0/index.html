场景二 : 版本回退

修改撤销

git checkout --<文件名>命令就是将<文件>在工作区的修改全部撤销，这里有两种情况：

第一种：<文件>被修改后没有git add提交至暂存区，执行git checkout则撤销本次所有修改。

第二种：<文件>已被git add提交至暂存区，然后又做了修改，执行git checkout则将<file>变为刚被提交至暂存区的状态。

简单的说，就是将这个文件回退至最近一次git add或git commit时的状态。

注意：如果你将文件误删了，且版本库内存在这个文件的提交，那么git checkout --<文件名>可以将这个误删的文件恢复。

简单的说，就是将版本库里的版本替换工作区的版本。

那如果想要回退到指定版本的话，先使用git log命令查看你所要回退的版本的版本号(commit ID)，然后使用git reset --hard <版本号>就可以了。

但回退之后觉得不好，又想撤销这次回退怎么办？使用git reflog命令查看一个每一次命令，然后再使用git reset命令恢复即可。

对于Git来说，回退操作只不过是将指向当前版本的HEAD指针指向另一个版本而已，所以很快很方便，不必担心崩溃情况，不过如果你胡乱更改.git文件夹的里文件就绝对会出错的。
