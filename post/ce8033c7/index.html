因Hoisting机制造成的麻烦可以规避，但ES6引入了块级作用域来让我们更舒服的控制变量的作用域。

块级声明

块级声明用于声明在指定块的作用域之外无法访问的变量。块级作用域存在于
1. 函数内部
2. 块中（ { }之间的区域 ）

let关键字

let关键字用来声明一个值可变的变量，用来替代var。

无法被提升

let关键字声明的变量会被限制在当前代码块中，因此开发者要将let声明写在封闭代码块的顶部。

禁止重声明

若当前作用域已存在某个标识符，再用let关键字声明会抛出错误。

循环中的函数let声明

let声明在每次迭代循环中都会创建一个新变量，并将其初始化为当前的值，从而使结果符合直觉预期。

const关键字

const关键字用来声明常量，其值一旦设定后便不可更改，因此const声明的变量必须初始化。

const关键字有以下的特点：

  1. 无法被提升
  2. 禁止重声明
  3. 不可以为const声明的常量再赋值
  4. 若绑定的值是对象，则对象里的值可以改变
  5. 无法在for循环里使用，但在for-in和for-of循环里与let行为一致
  6. 不可以在未被声明前访问（临时死区）

临时死区

因为不可被提升，所以let和const关键字声明的变量只能先被声明，才能被访问，不然会报错。
临时死区（temporal dead zone)：
Javascript引擎在发现变量声明时，要么提升到作用域顶部(var)，要么放入TDZ中(let,const)。访问TDZ中的变量会触发运行时错误，只有执行过变量声明语句后，才会从TDZ中放出。
